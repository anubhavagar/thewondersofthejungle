## Executive Summary
This audit applies advanced UX principles (**Fitts's Law**, **Hick's Law**, and **Biometric Feedback design**) to the existing FastAPI/MediaPipe gymnastics stack. The goal is to move beyond a technical pose-detection demo and create an elite, intuitive tool for competitive gymnasts.

## 1. Data Visualization: Intuition Over Information

### 1.1 The "Kinetic Heat" Gauge
*   **Principle**: Cognitive Load Reduction.
*   **Implementation**: Instead of raw text degrees (e.g., "172.4¬∞"), use circular "Safety Rings" around joints. 
*   **Visual Logic**: 
    *   **Acid Green**: Optimal range.
    *   **Neon Amber**: Warning/Inefficiency zone.
    *   **Pulse Red**: Immediate injury/form-break risk.
*   **Goal**: The gymnast should read the *color*, not the *number*.

### 1.2 Selective Landmark Density (Hick's Law)
*   **Principle**: Hick's Law (Time to make a decision increases with complexity).
*   **Implementation**: Don't show all 33 MediaPipe landmarks by default. 
*   **Logic**: Context-aware skeleton. If the user selects "Handstand," highlight the *Shoulder-Hip-Ankle* line. Fade out the arms and facial mesh to 10% opacity.

## 2. Contextual Awareness: Active vs. Review UI

### 2.3 Fitts's Law for Active Training (Video)
*   **Principle**: Fitts's Law (Target size and distance determine interaction speed).
*   **Implementation**: In "Active Camera" mode, UI elements must be "Mat-Friendly."
*   **Logic**: High-contrast, 80px+ target areas for "Start/Stop" and "Flag Moment." These should be positioned at the screen edges for easy thumb-reach if holding the phone, or visible from 5-10 feet away.

### 2.4 Scrub-to-Split (Image/Review)
*   **Principle**: Temporal Context.
*   **Implementation**: Automatic "Keyframe Extraction."
*   **Logic**: The UI should generate a "Phase Strip" showing the 3 critical moments of any skill: *Entry, Apex (Peak), and Recovery*. This eliminates manual scrubbing to find the error.

## 3. The 'Aha!' Moment: Visual Error Logic

### 3.5 The Red-Flash Joint (Biometric Feedback)
*   **Principle**: Instantaneous Error Correction.
*   **Implementation**: Real-time glitch effects.
*   **Logic**: If an angle breaks a safety threshold (e.g., knee collapse), the joint skeleton should "glitch" or flash red. 
*   **Impact**: The user doesn't check the data later; they *feel* the error as it happens on screen.

### 3.6 Apex Anchor Annotations
*   **Principle**: Instructional Precision.
*   **Implementation**: Auto-freezeframes with technical tags.
*   **Logic**: When the model detects the "Max Extension," drop a temporary tag like *"+5¬∞ Hip Flexion Needed"*. This is the digital equivalent of a coach's physical tap.

## 4. Professional Aesthetic: Elite Performance HUD

### 4.7 OLED HUD Performance Theme
*   **Style**: Dark Mode (Oled) + Cyberpunk HUD.
*   **Palette**: 
    *   **Background**: #000000 (True Black) for maximum focus.
    *   **Accents**: Cyan (#00FFFF) for data, Neon Amber (#FFB300) for alerts.
*   **Font**: Fira Code (Mono) for numbers/data, Figtree for descriptive text.

### 4.8 Glassmorphism Layering
*   **Implementation**: `backdrop-filter: blur(12px)`.
*   **Logic**: Data panels should feel like they are floating in mid-air over the gymnast. The athlete remains the "Source of Truth," and the data is a transparent layer of intelligence.

## 5. Indispensable Features

### 4.9 Voice-Activation (Hands-Free)
*   **Principle**: Environmental Constraints.
*   **Logic**: Implement voice triggers ("Capture", "Reset") so gymnasts don't have to break their "Health Ritual" or walk back to the device between reps.

### 4.10 "Consistency Trends" (Psychological Aha!)
*   **Principle**: Gamified Progress.
*   **Logic**: A simple sparkline showing hip-stability over the last 10 attempts. It proves that despite one bad rep, the *overall* trend is King-status.
